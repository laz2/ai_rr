Установка и настройка среды

разработки языка SCP

 

Известно, что для любого языка программирования необходима среда разработки – будь то компилятор или интерпретатор. Язык SCP не является исключением, поэтому перед тем, как приступить непосредственно к программированию, необходимо подготовить  пакет необходимых программ.

Необходимо выполнить следующую последовательность шагов:

1.     Установить модуль sc-core, инсталлятор последней версии которого можно взять из папки по адресу «\\Info\StudInfo\OSTIS_SOFT\kpm\sc-core». По умолчанию инсталлятор не добавляет путь к папке, где находятся исполняемые файлы, в переменную PATH. Желательно при установке указать, чтобы он модифицировал переменную PATH. Но инсталлятор в блюбом случае установит переменную среды SC_CORE_HOME, которая будет иметь в качестве значения путь к корневой директории sc-core. Причем из sc-core нам понадобятся следующие компоненты:

·        %SC_CORE_HOME%\bin\scs2tgf.exe - конвертор формата scs в tgf;

·        %SC_CORE_HOME%\bin\start-pm.exe – средство запуска процессорного модуля из консоли с указанным репозиторием;

·        %SC_CORE_HOME%\bin\m4scp2scs.bat – конвертор из m4scp в scs;

·        %SC_CORE_HOME%\examples\fs_repo_src – репозиторий исходных текстов с примером алгоритма поиска минимального пути

2.     Установить сборщик репозитория, архив с последней версией которого взять из папки по адресу «\\Info\StudInfo\OSTIS_SOFT\tools\repobuilder». Например, этот архив можно разархивировать в %SC_CORE_HOME%\repobuilder и прописать для удобства это папку в переменную PATH, выполнив команду «set PATH=%PATH%;%SC_CORE_HOME%\repobuilder». В билдере нас будут интересовать только один файл:

·        %SC_CORE_HOME%\repobuilder\builder.exe – бинарная версия сборщика репозитория.

На этом установка и первоначальная настройка среды разработки заканчивается.
