Класс операторов работы с процессами

 

Содержание главы

1.     Оператор запуска scp-программ callReturn

2.     Оператор завершения scp-программ return

 

 

Данный класс представлен операторами call, callReturn, waitReturn, return. Предназначены для запуска процесса (call), запуска программы (callReturn), ожидание завершения (waitReturn) и завершения процесса (return). Наиболее широкое применение имеют операторы callReturn и return.

 

Оператор запуска scp-программ callReturn

 

Синтаксис: 

сallReturn([1_: procedure_name,

            2_: {[1_: param1,

                  2_: param2,

                  3_: param3]}

           ])

 

Параметры:

procedure_name

[in] константный узел, который содержит путь к вызываемой процедуре

param1

[in, out] первый параметр вызываемой процедуры

param2

[in, out] второй параметр вызываемой процедуры

param3

[in, out] второй параметр вызываемой процедуры

proc_descr

[in, out] описатель (далее дескриптор) создаваемого процесса

Назначение:

Оператор callReturn предназначен для запуска scp-программ. Оператору, следующему после callReturn, управление передается только после завершения запускаемой scp-программы.

В качестве первого параметра оператора выступает константный узел scp-программы. Второй параметр – это упорядоченное множество параметров вызываемой scp-программы. При чем числовые атрибуты множества параметров в операторе и в вызываемой программе должны соответствовать. Каждый параметр оператора  callReturn уточняется атрибутом fixed_. Но параметры вызываемой программы уточняются только числовыми атрибутами. Возможен вызов scp-программы с не полным набором ее параметров (смотрите Пример 2).

 

Внимание: Для корректной работы необходимо, что бы вызываемая программа имела цельную последовательность числовых атрибутов параметров (1_, 2_, 3_ ... 5_). Не допустимой является ситуация 1_, 3_, 4_ и так далее.

Пример 1. Необходимо вызвать процедуру add, которая находится в файле add_content (каталоге proc). Процедура add имеет два параметра.

 

// В секции констант программы пишем

add_program =”proc/add_content/add”;

...

// Вызываем процедуру add

сallReturn([1_: fixed_: add_program,

            2_: fixed_: {[1_: param1,

                          2_: param2]}

           ])

В данном случае будет вызвана процедура add с параметрами param1 и param2. Параметры могут быть как входными, так и выходными.

Пример 2. Необходимо вызвать процедуру GetNext, которая находится в файле GetNext (каталоге proc). Процедура GetNext имеет три параметра.

 

// В секции констант программы пишем

add_program =”proc/GetNext/GetNext”;

...

// Вызываем процедуру add

сallReturn([1_: fixed_: add_program,

            2_: fixed_: {[1_: param1,

                          3_: param3]}

           ])

Пример демонстрирует вызов процедуры с определенными параметрами (в этом случае – это первый и третий параметры процедуры GetNext).

 

Оператор завершения scp-программ return

 

Синтаксис: 

return()

 

Параметры:

Оператор не имеет параметров

 

Назначение:

Оператор return предназначен для завершения scp-программ. После успешного выполнения оператора return происходит возврат управления в точку вызова данной процедуры. Так же после выполнения оператора формируются выходные параметры scp-программы. В некоторых случаях после оператора return следует ключевое слова языка end.

 

Оператор параллельного запуска scp-программ call

 

Оператор call представляет собой оператор запуска scp-программ, которые выполняются параллельно.

 

Оператор ожидания выполнения scp-процесса waitReturn

 

Оператор позволяющий ожидать выполнения определенного scp-процесса.
