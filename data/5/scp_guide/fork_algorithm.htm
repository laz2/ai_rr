Способы ветвления алгоритма scp-программы

 

Содержание главы

1.     Scp-аналог конструкции языка С if-then-else

2.     Scp-аналог конструкции языка С switch-case

3.     Scp-аналог конструкции языка С while

 

Так как единственно возможным средством управления порядком следования операторов являются метки, то все способы ветвления организованы на их основе (на основе использования меток).

 

 

 

Scp-аналог конструкции языка С if-then-else

 

Реализация конструкции if-then-else на языке SCP имеет ряд особенностей, связанных с использованием меток (смотрите раздел Метки).

 

Синтаксис конструкции if-then-else:

operator_scp([

                 1_: operand1,

                 2_: operand2,

                      ...

              ], goto_yes, goto_no)

 

label(goto_yes)

...

label(goto_no)

...

 

Параметры:

operator_scp

любой scp-оператор

goto_yes

имя метки перехода, в случае успешного выполнения оператора

goto_no

имя метки перехода, в случае безуспешного выполнения оператора

label(goto_yes)

метка перехода, в случае успешного выполнения оператора operator_scp осуществляется выполнение оператора следующего за данной меткой

label(goto_no)

метка перехода, в случае безуспешного выполнения оператора operator_scp осуществляется выполнение оператора следующего за данной меткой

 

Пример 1. Классический пример проверки условия.  

 

ifType([

         1_: arc_: fuz_: arc

        ], , no)

...

 

label(no)

printNl([1_: no_arc_msg], exit, exit)

 

...

 

label(exit)

return()

 

Как видно из примера, можно использовать только одну метку no, а в случае выполнения условия будет исполнятся оператор следующий за оператором проверки типа ifType. 

 

 

 

 

Scp-аналог конструкции языка С switch-case

 

Синтаксис конструкции switch-case:

operator_scp1([

                 1_: operand1,

                 2_: operand2,

                      ...

              ], goto_ case_1,)

 

operator_scp2([

                 1_: operand1,

                 2_: operand2,

                      ...

              ], goto_ case_2,)

 

...

 

operator_scpN([

                 1_: operand1,

                 2_: operand2,

                      ...

              ], goto_ case_N, goto_default)

 

label(goto_ case_1)

...

 

label(goto_ case_2)

...

 

label(goto_ case_N)

...

 

label(goto_default)

...

 

 

Параметры:

operator_scp1 …  operator_scpN

любые scp-операторы

goto_case_1 … goto_case_N

имена меток перехода, в случае выполнения соответствующих условий

goto_default

имя метки перехода, в случае не выполнения ни одного из условий

label(goto_case_1) … label(goto_case_N)

метки перехода, в случае выполнения условий операторов operator_scp, осуществляется выполнение операторов следующего за соответствующими метками

label(goto_default)

метка перехода, в случае не выполнения условия оператора operator_scp, осуществляется выполнение оператора следующего за данной меткой

 

 

 

Scp-аналог конструкции языка С while

 

Синтаксис конструкции while:

label(goto_yes_condition)

 

...

 

operator_scp([

                 1_: operand1,

                 2_: operand2,

                      ...

              ], goto_yes_condition,)

 

 

Параметры:

operator_scp

любой scp-оператор

goto_yes_condition

имя метки перехода, в случае выполнения условия

label(goto_yes_condition)

метка перехода, в случае выполнения условия оператора operator_scp, осуществляется выполнение оператора (тела цикла) следующего за данной меткой

 

Примечание:

В приведённом выше синтаксисе конструкции while происходит выполнение цикла, пока выполняется условие goto_yes_condition, однако возможно осуществлять цикл и по условию goto_no_condition и по условию goto_error_condition.
