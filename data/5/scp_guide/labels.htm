Метки, как средство управления

порядком следования операторов

 

Содержание главы

1.     Синтаксис меток

2.     Оператор безусловного перехода _goto

3.     Практические советы по использованию меток

 

Основным и единственным средством управления порядком следования операторов на момент написания данного руководства являются метки. Использование меток в языке SCP идентично использованию меток в других языках. Отличие заключается только в синтаксисе. Для объявления метки в языке присутствует специальное слово label. В общем случае объявление метки выглядит следующим образом:

 

Синтаксис:

label(label_name)

 

Параметры:

label_name

[in] имя метки

Назначение:

При переходе к метке с именем label_name осуществляется выполнение оператора следующего за этой меткой.

 

Внимание. Имя метки должно быть уникальным. В случае совпадения имен объявленных меток возникает конфликт (неоднозначность перехода).

Внимание. В рамках одного файла необходимо использовать уникальные метки, но в различных файлах допускается использование одинаковых имен меток.

Внимание. У любого scp-оператора метки могут либо отсутствовать, либо присутствовать в необходимом количестве. В случае отсутствия меток управление передаётся следующему по порядку оператору. Возможны следующие способы написания scp-операторов:

 

print([1_: node])

print([1_: node] ,,,)

print([1_: node] ,yes,,error)

print([1_: node] ,yes)

print([1_: node] ,,,error)

 

Пример 1. Использование меток. В зависимости от результата генерации выполняются определенные действия:

 

genEl([

         1_: assign_: node_: const_: node

      ], successfully_gen, unsuccessfully_gen, gen_error)

...

 

label(successfully_gen)

printEl([1_: node], exit, exit)

 

label(unsuccessfully_gen)

printNl([1_: unsuccessfully_gen_msg], exit, exit)

 

label(gen_error)

printNl([1_: error_msg], exit, exit)

 

...

 

label(exit)

return()

 

                                                           

Оператор безусловного перехода _goto

Синтаксис:

_goto(label_name)

 

Параметры:

label_name

[in] имя метки

 

Назначение:

Оператор осуществляет безусловный переход по метке с именем label_name.

 

Практические советы по использованию меток

1.       Желательно использовать длинные и максимально информативные имена меток, так как это позволит сократить число конфликтов, связанных с именами меток и повысит читабельность программы.

2.       Имена меток по возможности должны отражать смысл оператора (либо логического блока) следующего за данной меткой (смотрите пример №1).

3.       Метки, как и операторы, желательно комментировать (смотрите раздел «Комментарии»).

4.       При объявлении новой метки можно использовать поиск по документу для предотвращения дублирования имен меток.
