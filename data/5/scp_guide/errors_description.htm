Описание ошибок, возникающих

на этапе формирования репозитория

 

Нижеприведённые сообщения возникают в интерпретаторе DOS команд cmd.exe после запуска процесса формирования репозитория.

Для удобства определения возникшей ошибки в данном подразделе приняты следующие соглашения:

·        fs_repo_src/folder/file_name.m4scp – это имя текущего обрабатываемого файла исходного текста, вызвавшего сообщение, вместо folder будет написана последовательность вложенных папок, а вместо file_name.m4scp имя файла

·        D:/SCP/Framework/make_repo.rb – это путь к среде разработки языка SCP

·        _op1, _op2, _op3, ... – таким образом обозначаются порядковые номера операторов, вызвавших ошибку

·        N1, N2, N3, ... - таким образом обозначаются порядковые аргументов операторов, вызвавших ошибку

 

Сообщение об ошибке

Пояснение

unexpeted TOK_NUM_CONT

неправильный синтаксис scs-конструкции

processing fs_repo_src/folder/file_name.m4scp  

yyerror: syntax error, unexpected TOK_IDENTIFIER, expecting '}' at (stdin) line number 259

parse error occured

в 259 строке файла file_name.m4scp возможно отсутствует запятая перед меткой в операторе

 

processing fs_repo_src/folder/file_name.m4scp

yyerror: syntax error, unexpected '}', expecting ';' at (stdin) line number 131

parse error occurred

в файле file_name.m4scp неправильно закомментирован исходный текст: в случае написания исходного текста m4scp необходимо обязательно комментировать операторы при помощи символов "#" или "//#" (требование конвертора m4)

processing fs_repo_src/folder/file_name.m4scp

yyerror: syntax error, unexpected TOK_IDENTIFIER, expecting ';' at ./../../include/scp_syn_file_name.scsy line number 114

parse error occured

в файле file_name.m4scp объявлен файл синонимов scp_syn_file_name.scsy, в котором отсутствует ";" после объявления узла

processing fs_repo_src/folder/file_name.m4scp

cannot open file ./../include/scp_syn_file_name.scsy: No such file or directory included from (stdin):18

 

в файле file_name.m4scp объявлен несуществующий файл синонимов scp_syn_file_name.scsy, либо имя файла синонимов неправильно задано

processing fs_repo_src/folder/file_name.m4scp

yyerror: syntax error, unexpected $undefined at (stdin) line number 6

parse error occured

если файл file_name.m4scp находится в начальной стадии разработки и в нём написаны только комментарии, то следует внести имя этого файла в список игнорируемых для формирования репозитория

 

Описание сообщений

интерпретатора языка SCP

 

В данном подразделе описаны сообщения, возникающие в интерпретаторе DOS команд cmd.exe после запуска утилиты start-pm.

Для удобства определения возникшей ошибки в данном подразделе приняты следующие соглашения:

·        operator – это идентификатор оператора, вызвавшего сообщение

 

Сообщение об ошибках

Пояснение

Cannot find next operator (for operand). Check if your next operator is constant node

Your program(/folder/file/node) haven't constants

 

данная ошибка возникает, когда scp-оператор содержит переходы на несуществующие метки. Такая ситуация возникает, когда в программе отсутствует оператор return()

start-pm.exe -d –run-program “/path/to/program” 1>_out_log

Actual parameter haven't value

ROLLBACK ERROR at line 1094

Failed to construct input params

 

возникает, когда происходит попытка наполнить неопределённым значением входной параметр подпрограммы, то есть в качестве параметра передаётся переменная, которая не была прежде определена

start-pm.exe -d –run-program “/path/to/program” 1>_out_log

Program parameter is output and matching actual parameter isn't variable

ROLLBACK ERROR at line 1113

Failed to construct input params

Actual parameter haven't value

ROLLBACK ERROR at line 1113

Failed to construct input params

 

при использовании константного узла в качестве выходного параметра подпрограммы происходит попытка изменить содержимого константного узла, а это недопустимо

 

start-pm.exe -d –run-program “/path/to/program” 1>_out_log

Type of operator scpAnalyzer_op98 incorrect

 

в случае использования некоторых scp-операторов необходима предварительная регистрация, в противном случае возникает данное сообщение об ошибке

start-pm.exe -d –run-program “/path/to/program” 1>_out_log

fixed_ operand haven't value

fixed_ operand haven't value

 

у оператора из класса searchSet неправильный шаблон поиска (например, отсутствуют зафиксированные элементы)

start-pm.exe -d –run-program “/path/to/program” 1>_out_log

Your program(/folder/filename/node) haven't constants

 

попытка с помощью оператора callReturn() запустить узел, не являющийся главным узлом программы (то есть узел, не содержащийся во множестве scpPrograms)

 

Сообщение о внутренних ошибках (internal error)

Пояснение

Message = "/This is text/"

printNl([1_: Message])

ошибка возникла из-за того, что текстовое сообщение было оформлено как путь к узлу, следует инициировать текстовый узел так: Message = /"This is text"/

...

Counter =n= 6

...

Counter =n= 2

...

 

нельзя переопределять значение константы несколько раз, так как scp-программа – это описание графа, а повторная инициализация приводит к конфликту

 

 

 

 

Описание сообщений протокола

работы программы

 

В данном подразделе описаны сообщения протокола работы программы (подробнее смотрите раздел «Протокол работы»).

Для удобства определения возникшей ошибки в данном подразделе приняты следующие соглашения:

·        programname – это имя выполняемой программы

·        operator – это идентификатор оператора, вызвавшего сообщение

·        _op1, _op2, _op3, ... – таким образом обозначаются порядковые номера операторов, вызвавших ошибку

·        N1, N2, N3, ... - таким образом обозначаются порядковые аргументов операторов, вызвавших ошибку

 

Информационные сообщения

Пояснение

нормальное выполнение оператора operator в scp-программе programname

Process 0235C478 is dying now

 

сообщение о завершении выполнения scp-процесса

Stopping process due to ERROR_STATE

принудительное завершение scp-процесса по причине возникновения критической ошибки

autosegment is 0235C688

установка автосегмента для выполнения запущенного scp-процесса

Program parameter 4 does not have matching actual parameter

 

вызываемой программе не передан выходной параметр 4.

Именно выходной, так как отсутствие входных параметров приведёт к ошибке

operator: Value N1 is not assigned

у оператора из класса search отсутствует зафиксированный аргумент (с атрибутом fixed_ )

autosegment is 0308BF98

call failed!

выходной параметр является константой (то есть не возможно вернуть значение параметра)

у оператора из класса searchSet зафиксированный первый аргумент (с атрибутом fixed_) не имеет содержимого

Executing programname_op4(searchSetStr3) of prg programname(03E93CC8)

Stopping process due to ERROR_STATE

Process 03E93CC8 is dying now

Например:

searchSetStr3([

                 1_:  fixed_: var1,

                 2_: assign_: var2,

                 3_: assign_: var3,

                 set1_: assign_: Var

               ])

Executing programname_op14(searchSetStr5) of prg programname(0407B540)

Stopping process due to ERROR_STATE

Process 0407B540 is dying now

 

у оператора из класса searchSet неправильный шаблон поиска (например, отсутствуют зафиксированные элементы)

Loading fs segment programname1/programname1

Warning: link target segment /programname2/programname2 wasn't found

в programname1 подключается файл scp-синонимов *.scsy, в котором присутствует несуществующий узел programname2

Executing programname_op20(operator) of prg programname(03F2A968)

Your operand variable_name is neither constant nor variable

Failed to fill operand info for operand N1

Stopping process due to ERROR_STATE

Process 03F2A968 is dying now

если в протоколе возникает ошибка после оператора operator с некоторым порядковым номером, а в действительности этому порядковому номеру соответствует оператор другого типа (не operator), то ошибка в неправильных метках. Например, в одном файле находятся несколько программ с одинаковыми метками. Один из методов наверняка избежать этого – в метки вносить имена программ (они всегда уникальны): label_name_in_program_name. Кроме того, это позволяет при взгляде на сплошной код определить к какой подпрограмме относится данный код, причём, не ища заголовка программы.
