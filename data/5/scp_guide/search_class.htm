Класс операторов ассоциативного поиска search

 

Содержание главы

1.     Оператор ассоциативного поиска трёхэлементной конструкции searchElStr3

2.     Оператор ассоциативного поиска пятиэлементной конструкции searchElStr5

3.     Оператор ассоциативного поиска трёхэлементной конструкции searchSetStr3 с автоматическим наполнением результирующих множеств

4.     Оператор ассоциативного поиска пятиэлементной конструкции searchSetStr5 с автоматическим наполнением результирующих множеств

 

Данный класс операторов предназначен для ассоциативного поиска элементов в памяти SCP машины по заданному шаблону.

 

Внимание: Хотя бы один операнд оператора должен быть отмечен атрибутом fixed_ (все остальные могут  отмечаться атрибутами assign_).

 

Чем больше операндов отмечены атрибутом  fixed_, тем уже область поиска. Все операнды, отмеченные assign_, после успешного выполнения оператора ассоциативного поиска становятся известны, то есть их значения можно использовать (например, для поиска). Операторы, которые в составе своего названия имеют Set (searchSetStr3, searchSetStr5), позволяют сформировать множество из указанных значений операндов. Более подробно каждый из операторов поиска рассмотрим отдельно.

Оператор ассоциативного поиска трёхэлементной конструкции searchElStr3

 

Синтаксис:

searchElStr3([

                 1_: node,

                 2_: arc,

                 3_: element

              ], goto_yes, goto_no, goto_error)

 

Параметры:

node

[in, out] узел поиска, который является первым элементом в трёхэлементной конструкции

arc

[in, out] дуга поиска, которая является вторым элементом в трёхэлементной конструкции

element

[in, out] элемент поиска, который является третьим элементом в трёхэлементной конструкции

goto_yes

метка успешного выполнения, если конструкция найдена

goto_no

метка безуспешного выполнения, если конструкция не найдена

goto_error

метка ошибочного выполнения, если в ходе поиска конструкции возникла ошибка

 

Назначение:

Данный оператор применяется для поиска элементов связанных между собой дугой. В качестве примера может служить поиск элемента во множестве, к которому он принадлежит. Оператор  searchElStr3 можно использовать и для поиска множеств, в которые входит данный элемент.

Пример 1. Задано не пустое множество set. Множество содержит в себе элементы elem1, elem2, elem3. Необходимо найти любой элемент множества.

searchElStr3([

                 1_: fixed _: node_ : set,

                 2_: assign_: pos_: arc_:  arc,

                 3_: assign_: node_: node

              ], yes, no)

 

В этом примере мы впервые применяем метки. Переход по метке yes осуществляется при условии успешного выполнения оператора поиска. Это значит, что найден любой элемент множества set, значение этого элемента будет присвоено переменной node. После этого можно использовать переменную node в качестве поиска по выходящим из нее дугам (т.е. если node так же является множеством). При безуспешном поиске (элемент не найден) переход осуществляется по метке no. Т.е. множество set не имеет элементов. При помощи searchElStr3 можно проверять пусто ли множество.

(1_) Первый параметр оператора отмечен атрибутом fixed_ так, как значение операнда известно. То есть  мы знаем, в каком множестве необходимо производить поиск элемента. Атрибут node_ говорит о том, что set это узел. Впрочем, его можно и не добавлять, так как в качестве первого операнда всегда является узел, следовательно, он может уточняться атрибутами const_ и var_ (константный и переменный узел)

(2_) В качестве второго параметра, в случае трёхэлементной конструкции всегда выступает дуга (по этой причине атрибут arc_ можно было и не добавлять). Дуга может быть уточнена как const_, так и var_ (константной и переменной). Помимо const_ и var_, дуга может уточняться атрибутами pos_ (позитивная, направлена от set к node) и neg_ (негативная, направлена от node к set). Необходимо учитывать тот факт, что константные дуги выходят только из константных узлов.

(3_) В примере третий элемент представляет собой узел. Поэтому он уточняется  атрибутом node_. Но в случае трёхэлементной конструкции операнд может быть представлен дугой, узлом и элементом неопределенного типа. И в зависимости от того, чем элемент представлен,  он уточняется соответствующими  атрибутами.

 

Пример 2. К примеру, есть элемент (node). Необходимо определить в состав, каких множеств он входит.

 

searchElStr3([

                 1_: assign _: set,

                 2_: assign_: neg_: arc_:  arc,

                 3_: fixed_: node_: node

              ])

 

После успешного выполнения оператора  переменная set будет являться знаком одного из множеств, в которые входит элемент node.

 

Оператор ассоциативного поиска пятиэлементной конструкции searchElStr5

 

Синтаксис: 

searchElStr5([

                1_: node,

                2_: arc,

                3_: element,

                4_: attr_arc,

                5_: attr_node

              ], goto_yes, goto_no, goto_error)

 

 

Параметры:

node

[in, out] узел поиска, который является первым элементом в пятиэлементной конструкции

arc

[in, out] дуга поиска, которая является вторым элементом в пятиэлементной конструкции

element

[in, out] элемент поиска, который является третьим элементом в пятиэлементной конструкции

attr_arc

дуга поиска, которая является четвёртым элементом в пятиэлементной конструкции

attr_node

узел поиска, который является пятым элементом в пятиэлементной конструкции

goto_yes

метка успешного выполнения, если конструкция найдена

goto_no

метка безуспешного выполнения, если конструкция не найдена

goto_error

метка ошибочного выполнения, если в ходе поиска конструкции возникла ошибка

 

Назначение:

Данный оператор применяется для поиска пятиэлементных конструкций. Для операндов оператора справедливы атрибуты, которые могут принимать соответствующие элементы пятиэлементной конструкции. В качестве примера может служить поиск элемента во множестве, который уточняется атрибутом.

 

Пример 3. Множество set  содержит элементы elem1, elem2, elem3, которые уточняются атрибутами attr1, attr2, attr3 соответственно. Необходимо найти любой из элементов множества.

searchElStr5([

                 1_: fixed _: const_: set,

                 2_: assign_: const_: pos_: arc,

                 3_: assign_: node_ : node,

                 4_: assign_: const_: pos_: attr_arc,

                 5_: assign_: const_: attr

              ], yes, no)

 

Переход по метке yes осуществляется при условии успешного выполнения оператора поиска. Это значит, что найден любой элемент множества set, значение этого элемента будет присвоено переменной node. Аналогично со всеми операндами, отмеченными атрибутом assign_ (их значения будут присвоены соответствующим переменным). При безуспешном поиске (элемент не найден) переход осуществляется по метке no. Т.е. множество set не имеет элементов с атрибутами.

(1_) Первый параметр оператора отмечен атрибутом fixed_ так, как значение операнда известно. То есть  множество, в котором производиться поиск известно.

(2_) В качестве второго параметра, в случае пятиэлементной конструкции всегда выступает дуга (по этой причине атрибут arc_ не пишем). Дуга может быть уточнена как const_, так и var_ (константной и переменной). Помимо const_ и var_ , дуга может уточняться атрибутами pos_ (позитивная, направлена от set к node) и neg_ (негативная, направлена от node к set). В приведенном примере производим поиск константной позитивной дуги. Необходимо учитывать, что константные дуги выходят только из константных узлов.

 

(3_) В примере третий элемент представляет собой узел. Поэтому он уточняется  атрибутом node_. Но в случае пятиэлементной конструкции операнд может быть представлен дугой, узлом и элементом неопределенного типа. И в зависимости от того, чем элемент представлен,  он уточняется соответствующими  атрибутами.

(4_) Четвертый элемент поиска – элемент атрибута, а именно дуга. Для поиска такого элемента можно использовать все допустимые для дуги атрибуты.

(5_) Пятый элемент поиска – элемент атрибута, а именно узел. Для поиска такого элемента можно использовать все допустимые для узла атрибуты.

 

Пример 4. К примеру, нам известен атрибут (attr). Необходимо выяснить, что уточняет данный атрибут и в состав, какого множества входит элемент, который уточняется данным атрибутом.

 

searchElStr5([

                 1_: assign_:  set,

                 2_: assign_:  pos_: arc,

                 3_: assign_: node_ : node,

                 4_: assign_: pos_: attr_arc,

                 5_: fixed _:  attr

              ])

 

После успешного выполнения оператора, данный в условии задачи атрибут attr будет уточнять node во множестве set, где set одно из множеств в которые входит  node.

 

Оператор ассоциативного поиска трёхэлементной конструкции searchSetStr3 с автоматическим наполнением результирующих множеств

 

Синтаксис: 

searchSetStr3([

                1_: node,

                2_: arc,

                3_: element,

                set1_: resultSet1,

                set2_: resultSet2,

                set3_: resultSet3

               ], goto_yes, goto_no, goto_error)

 

 

Параметры:

node

[in, out] узел поиска, который является первым элементом в трёхэлементной конструкции

arc

[in, out] дуга поиска, которая является вторым элементом в трёхэлементной конструкции

element

[in, out] элемент поиска, который является третьим элементом в трёхэлементной конструкции

resultSet1

[out] множество, наполняемое первыми элементами найденных трехэлементных конструкций

resultSet2

[out] множество, наполняемое вторыми элементами найденных трехэлементных конструкций

resultSet3

[out] множество, наполняемое третьими элементами найденных трехэлементных конструкций

goto_yes

метка успешного выполнения, если множество наполнено

goto_no

метка безуспешного выполнения, если множество не наполнено

goto_error

метка ошибочного выполнения, если в ходе поиска конструкции возникла ошибка

 

Назначение:

Данный оператор предназначен для поиска и наполнения найденными элементами множеств. То есть атрибут set1_ говорит о том, что множество resultSet1 будет наполнена элементами node. Перед наполняемым множеством (resultSet1, ... , resultSet3) необходимо наличие атрибутов fixed_ либо assign_. В случае fixed_ наполняемое множество будет дополнено. А в случае assign_ множество сформируется заново. Оператор searchSetStr3 удобно использовать для создания копий множеств, так как в большинстве случаев необходимо обрабатывать не сами множества, а их копии.

 

Пример 5. Множество set  содержит элементы elem1, elem2, elem3. Необходимо создать копию множества set.

searchSetStr3([

                 1_: fixed _: const_: set,

                 2_: assign_: const_: pos_: arc,

                 3_: assign_: node_ : node,

                 set3_: assign_: set_copy

               ], yes, no)

 

Переход по метке yes осуществляется при условии успешного выполнения оператора searchSetStr3. Это значит, что элементы множества set, будут принадлежать и множеству set_copy, то есть будут сформированы дуги, выходящие из узла set_copy и входящие в узлы элементов (elem1, ... , elem3).

Пример 6. Множество set_a  содержит элементы elem1, elem2, elem3. Множество set_b содержит дуги arc1, arc2. Необходимо добавить во множество set_b дуги принадлежности узлов elem1, ... , elem3 к узлу set_a.

searchSetStr3([

                 1_: fixed _: const_: set_a,

                 2_: assign_: const_: pos_: arc,

                 3_: assign_: node_ : node,

                 set2_: fixed_: set_b

               ], yes, no)

 

После успешного выполнения оператора множество set_b будет выглядеть следующим образом:

 

Оператор ассоциативного поиска пятиэлементной конструкции searchSetStr5 с автоматическим наполнением результирующих множеств

 

Синтаксис:

                set1_: resultSet1,

element

attr_arc

[in, out] дуга поиска, которая является четвёртым элементом в пятиэлементной конструкции

attr_node

[in, out] узел поиска, который является пятым элементом в пятиэлементной конструкции

resultSet1

[out] множество, наполняемое первыми элементами найденных пятиэлементной конструкций

resultSet2

[out] множество, наполняемое вторыми элементами найденных пятиэлементной конструкций

resultSet5

[out] множество, наполняемое третьими элементами найденных пятиэлементной конструкций

goto_yes

метка успешного выполнения, если множество наполнено

goto_error

метка ошибочного выполнения, если в ходе поиска конструкции возникла ошибка

Перед наполняемым множеством (resultSet1, ... , resultSet5) необходимо наличие атрибутов fixed_ либо assign_. В случае fixed_ наполняемое множество будет дополнено.                  5_: assign_: const_: attr,

                 set4_: assign_: attr_arc_set

               ], yes, no)
