Класс операторов проверки условий if

 

Класс операторов if предназначен для определения истинности высказываний.

 

Оператор проверки типа элемента ifType

Синтаксис: 

ifType([

            1_: element

         ], goto_yes, goto_no, goto_error)

 

Параметры:

element

[in] элемент проверки (уточняться атрибутами, на основе которых

проводится проверка)

goto_yes

метка перехода, если операнд удовлетворяет заданному условию

goto_no

метка перехода, если операнд не удовлетворяет заданному условию

goto_error

метка ошибочного выполнения, если в ходе проверки условия возникла ошибка

 

Назначение:

Оператор осуществляет проверку типа элемента (операнда). Тип проверяется по признаку: переменная, константа. Так же по признаку: узел, элемент неопределенного типа, дуга. В случае дуги: позитивная, негативная, не четкая. Помимо проверки типа, проверяется, вычислено ли значение операнда (если операнд оператора - переменная)

 

Внимание: Операнд можно помечать атрибутом fixed_, но его значение перед выполнением оператора должно быть найдено. Иначе возникнет ошибка, так как атрибутом fixed_ можно помечать только операнд с найденным значением.

Пример 1. Необходимо определить, является ли arc нечеткой дугой.

 

ifType ([

             1_: arc_: fuz_: arc

         ],yes,no)

 

Если arc нечеткая дуга, то переход по метке yes.

Если условие для arc не выполняется (то есть arc не дуга или не нечеткая дуга) либо значение arc не известно, то переход по метке no.

 

 

Оператор проверки равенства содержимого двух элементов  ifEq

Синтаксис: 

ifEq([

        1_: element_1,

        2_: element_2

       ], goto_yes, goto_no, goto_error)

 

Параметры:

element_1

[in] первый элемент проверки, его содержимое сравнивается с содержимым второго элемента

element_2

[in] второй элемент проверки, его содержимое сравнивается с содержимым первого элемента

goto_yes

метка перехода, если содержимое операндов совпадает

goto_no

метка перехода, если содержимое операндов не совпадает

goto_error

метка ошибочного выполнения, если в ходе проверки условия возникла ошибка

 

Назначение:

Оператор сравнивает содержимое двух операндов. В качестве операндов данного оператора могут выступать как переменные, так и константы. Содержимым операндов могут являться и числа (числовое содержимое), и строки (строковое содержимое).

 

Внимание: Для корректной работы оператора ifEq необходимо, что бы значения (не путать с содержимым) сравниваемых операндов были найдены.

Пример 2. Необходимо сравнить содержимое node1 и node2.

 

ifEq ([

         1_: fixed_: node1,

         2_: fixed_: node2

       ],yes,no)

 

Если содержимое node1 и node2 совпадает, то переход по метке yes. Если не совпадает, то переход по метке no.

 

Оператор проверки равенства значений двух элементов  ifCoin

Синтаксис: 

ifCoin([

            1_: element_1,

            2_: element_2

         ], goto_yes, goto_no, goto_error)

 

 

Параметры:

element_1

[in] первый элемент проверки, его значение сравнивается со значением второго элемента

element_2

[in] второй элемент проверки, его значение сравнивается со значением первого элемента

goto_yes

метка перехода, если значения операндов совпадает

goto_no

метка перехода, если значения операндов не совпадает

goto_error

метка ошибочного выполнения, если в ходе проверки условия возникла ошибка

 

Назначение:

Действия данного оператора аналогичны действиям оператора  ifEq, за исключением того, что сравниваются значения (не содержимое) рассматриваемых операндов.

 

Внимание: Значения операндов должны быть найдены. И, как следствие, отмечаются атрибутом fixed_.

Внимание: По крайней мере, один из операндов должен быть переменной. Так как сравнивать две константы нет смысла.

 

Оператор проверки строгого неравенства содержимого двух элементов  ifGr

 

Синтаксис: 

ifGr([

        1_: element_1,

        2_: element_2

       ], goto_yes, goto_no, goto_error)

 

 

Параметры:

element_1

[in] первый элемент проверки, его содержимое сравнивается с содержимым второго элемента

element_2

[in] второй элемент проверки, его содержимое сравнивается с содержимым первого элемента

goto_yes

метка перехода, если содержимое первого операнда больше

содержимого второго операнда

goto_no

метка перехода, если содержимое первого операнда меньше

содержимого второго операнда

goto_error

метка ошибочного выполнения, если в ходе проверки условия возникла ошибка

 

Назначение:

Оператор позволяет сравнивать  содержимое (в случае данного оператора целесообразно сравнивать числовое содержимое) рассматриваемых операндов.

 

Внимание: Значения операндов должны быть найдены. И, как следствие, отмечаются атрибутом fixed_. Если значение одного из рассматриваемых операндов будет не вычислено, то произойдет переход по метке  goto_no.

ifFormCont([

Параметры:

goto_no

Оператор проверки вычислено ли значение элемента ifVarAssign

Синтаксис: 

ifVarAssign([

                1_: element

              ], goto_yes, goto_no, goto_error)

 

 

Параметры:

element

[in] элемент проверки, как правило переменная (может не уточнятся атрибутами)

goto_yes

метка перехода, если значение переменной извечно

goto_no

метка перехода, если значение переменной не найдено

goto_error

метка ошибочного выполнения, если в ходе проверки условия возникла ошибка

 

Назначение:

Оператор проверяет условие: вычислено ли значение переменной, соответствующей значению операнда. ifVarAssign удобно применять для проверки на корректность входных параметров программы.

 

 

Пример 3. Необходимо проверить корректность входных значений программы. param – входной параметр программы.

 

ifVarAssign ([

       1_: param

     ],,exit)

 

Если значение param не известно, то переход по метке exit. Если известно, то переход к оператору, следующему прямо за оператором проверки  ifVarAssign.
