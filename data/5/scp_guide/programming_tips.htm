Советы по программированию

 

1.     Список ключевые узлов,  которые могут помочь при написании scp-программ:

programSCP – это узел, который содержит идентификаторы всех scp-программ.

operatorSCP – это узел, который содержит узлы всех scp-операторов.

 

2.     Если из элемента неопределённого типа выходит (в процессе генерации, например) дуга, то элемент автоматически превращается в узел.

 

3.     searchSetStr5([1_:  fixed_: SET1,

                   2_: assign_: arc,

                   3_: assign_: node,

                   4_: assign_: arc2,

                   5_:  fixed_: attr_,

                   set3_: assign_: SET3])

 

После выполнения этого оператора, если будут найдены конструкции, удовлетворяющие шаблону, они будут добавлены в SET3, но node может быть неопределённым. Поэтому одновременно две задачи невозможно выполнить. Это важно, так как такая ситуация часто возникает для оптимизации в циклах, когда производится попытка исключить дополнительный scp-оператор класса select().

 

4.     В отдельную программу надо выносить либо обособленные по смыслу алгоритмы, либо очень объёмные (для упрощения отладки), либо некоторые сходные участи алгоритмов для совместного использования. Небольшие подпрограммы желательно объединять в одну программу (по возможности) для исключения накладных расходов памяти (дополнительная подпрограмма содержит много "сервисных(служебных)" узлов), кроме того, для исключения дополнительных вычислений, так как создание процесса - сложный многошаговый процесс.

 

5.     При совмещении нескольких программ в один файл можно руководствоваться следующим правилом: если подпрограмма вызывается только одной родительской программой, то совместно использовать её не придётся – имеет смысл написать подпрограмму в файле программы, тогда просмотр программы и её подпрограммы не будет прерываться переключением файлов. Это повышает читаемость, позволяет просматривающему чётче понимать взаимосвязи программ и помогает структурировать большое количество файлов крупного проекта с целью уменьшения количества файлов.
