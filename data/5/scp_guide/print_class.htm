Класс операторов вывода информации в протокол print

 

Содержание главы

1.     Оператор вывода информации в протокол print

2.     Оператор вывода информации в протокол printNl

3.     Оператор вывода информации в протокол printEl

 

Данный класс предназначен для непосредственного вывода информации в файл протоколирования. В протокол может выводиться информация как о элементе (окрестность элемента по входящим и выходящим дугам), так и о его содержимом.

Оператор вывода информации в протокол print

 

Синтаксис: 

print([

        1_: element

       ], goto_yes, goto_no, goto_error)

 

Параметры:

element

[in] элемент вывода (может не уточняться атрибутами)

goto_yes

метка успешного выполнения

goto_no

метка безуспешного выполнения

goto_error

метка ошибочного выполнения, если в ходе вывода информации возникла ошибка

 

Назначение:

Действия оператора print аналогичны действиям оператора printNl (смотрите ниже), за исключением того, что printNl переводит каретку на следующую строку в файле протоколирования.

 

Оператор вывода информации в протокол printNl

Синтаксис: 

printNl([

            1_: element

         ], goto_yes, goto_no, goto_error)

 

 

Параметры:

element

[in] элемент вывода (может не уточняться атрибутами)

goto_yes

метка успешного выполнения

goto_no

метка безуспешного выполнения

goto_error

метка ошибочного выполнения, если в ходе вывода информации возникла ошибка

 

Назначение:

Данный оператор позволяет вывести в файл протоколирования содержимое (строковое или числовое) операнда. Обязательным атрибутом является 1_. Все остальные атрибуты являются не обязательными.

 

Внимание: Операнд, перед выводом в протокол должен быть найден.

Пример 1. Необходимо вывести содержимое узла node.

 

printNl([

    1_: node

         ])

 

Оператор вывода информации в протокол printEl

 

Синтаксис: 

printEl([

            1_: element

         ], goto_yes, goto_no, goto_error)

 

 

Параметры:

element

[in] элемент вывода (может не уточняться атрибутами)

goto_yes

метка успешного выполнения

goto_no

метка безуспешного выполнения

goto_error

метка ошибочного выполнения, если в ходе вывода информации возникла ошибка

 

Назначение:

Данный оператор позволяет вывести в файл протоколирования информацию об операнде, а именно его окрестность по входящим и выходящим дугам. Обязательным атрибутом является 1_. Все остальные атрибуты являются не обязательными.

 

Внимание: Операнд, перед выводом в протокол должен быть найден.

Пример 1. Необходимо вывести информацию об узле set.

 

printEl([

             1_: set

         ])
